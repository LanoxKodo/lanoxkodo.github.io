What kind of data is collected

The following are what TigerGuard uses and stores in some manner, all of this are explictitly as written here, the bot will never go beyond scope that it does not have to.

- Discord User IDs
- Discord Server IDs
- Temporary Data

Below is the scope to each of the above and how each are used.

Discord User IDs

TigerGuard stores the UUID of users provided by Discord API for use in: 1) Guild-specific Leveling Systems, 2) Guild-agnostic Leveling System (to eventually supercede the Guild-specific ones), 3) Guild Poll responses for voting.

Note: For things like the guild polls and the xp systems, your UUID is not stored until you actually partake in using the bot after the associated server sets up the required opt-in features, after that, you sending a message in a channel will then set you up in the xp system if the xp system was configured, or your ID will be recorded with a vote value in the poll table which itself is a limited-time table which then will print a message into the channel where the vote was created, and after that TigerGuard will delete the associated table for that particular poll as it no longer will be utilized. The poll system is set up this way so you may switch your vote while the vote is up, but then lock the vote results and calculate the majority decision while hiding your vote the entire time unlike Discords poll system which shows who voted for what option.

TigerGuard does NOT share the UUID of users with any 3rd parties besides Discord when using their API calls.

Discord Server IDs

TigerGuard stores the UUID of servers provided by Discord API for use in: 1) Guild-specific Leveling Systems, 2) Guild Polls, 3) Guild Level Roles, which relate to the leveling system(s), 4) Opt-In data that must be specified by server admins, and also may be revoked at any time.

TigerGuard does NOT share the UUID of servers with any 3rd parties besides Discord when using their API calls.

Temporary Data

TigerGuard stores misc data that does not live permantently in the bot database. This data is can be as simple as "Yes"/"Abstain"/"Nay" vote values for guild polls. The data can also include when a user joined a call and give them xp after leaving so long as the time spent in call was 15 minutes, the data is then wiped after the xp is awarded and stored in the associated xp system.

TigerGuard does NOT share any data in this category with anyone, it strictly is blind in the sense no one ever sees it besides myself if I have to make a backend adjustment in the database itself as part of bug tracking.

Overall

As noticed above, there isn't anything that is really forced onto you, the user or server. Any data is used in a very confined way and never released for any groups beyond Discord itself for typical API usage.

If you wish to have your data removed and the bot is having trouble removing something. Please contact me either in my support server as mentioned near the start. Should there be any problems, I want to ensure you that it will be my utmost priority to work with you on an issue so requests are not ignored.
